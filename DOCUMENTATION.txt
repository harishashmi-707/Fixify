================================================================================

Project Name    : Fixify Pakistan
Version         : 2.0.0
Type            : Web-Based Home Services Booking Platform
Domain          : On-Demand Service Marketplace
Target Region   : Pakistan (Islamabad, Lahore, Karachi, Rawalpindi)
Currency        : PKR (Pakistani Rupee)

Description:
Fixify Pakistan is a full-stack web application that connects customers
with verified technicians for home services such as mobile repair, plumbing,
electrician work, AC services, laptop repair, and more. Customers can browse
services, view technician profiles, book appointments, make payments, leave
reviews, and communicate with technicians through the platform.

5.2 Design Pattern:
The application follows a procedural PHP architecture with MVC-inspired
organization. Pages are served through a router that maps clean URLs to
PHP files. Shared logic is centralized in includes/ and app/ directories.

5.3 Frontend, Backend, and Database Interaction (Data Flow):
The system operates on a classic Client-Server request-response cycle, seamlessly
integrating the frontend (UI), backend (PHP), and MySQL database:

1. Frontend (Client Side):
   - Built with HTML, CSS (Bootstrap 5), and Vanilla JavaScript.
   - When a user interacts with the UI (e.g., submitting a booking form),
     JS handles client-side validation, enhances the UI with animations (AOS), 
     and formats the request for the backend.
   - Asynchronous requests (AJAX/Fetch) are utilized in specific areas 
     to dynamically load data without full page reloads.

2. Backend (Server Side):
   - The server receives the HTTP request and routes it via `router.php` 
     to the appropriate PHP script (e.g., `book-service.php`).
   - The PHP script performs crucial server-side validation, validates CSRF 
     tokens, and checks user authentication/authorization.
   - It processes core business logic, such as calculating service costs,
     applying commissions, or dispatching notifications.

3. Database Management (MySQL):
   - The backend communicates securely with the MariaDB/MySQL database using 
     PDO (PHP Data Objects) and prepared statements to prevent SQL injection.
   - Data is queried, inserted, or updated based on the request (e.g., 
     creating a new record in the `bookings` table).
   - Custom wrapper functions (`dbQuery()`, `dbQueryOne()`, `dbExecute()`) 
     abstract and streamline these PDO interactions.

4. Response & Rendering:
   - Upon successful database operations, the backend fetches the required 
     data (utilizing the file-based caching system for performance optimization).
   - This data is injected into HTML templates (views).
   - The generated HTML, along with success/error flash messages (rendered 
     via SweetAlert2), is sent back to the browser for the user to see.

8. SECURITY FEATURES
================================================================================

8.1 Authentication & Authorization:
- Bcrypt password hashing (password_hash / password_verify)
- Role-based access control (user, technician, admin)
- Session-based authentication with regeneration on login
- JWT token authentication for REST API
- Login attempt tracking with automatic lockout
- Secure session configuration (httponly, samesite)

8.2 Input Protection:
- CSRF token protection on all forms
- Prepared statements (PDO) for all database queries (SQL injection prevention)
- Input sanitization with htmlspecialchars (XSS prevention)
- Email and phone validation
- File upload validation (type, size, content scanning)

8.3 HTTP Security Headers:
- Content-Security-Policy (CSP)
- X-Frame-Options: SAMEORIGIN (clickjacking prevention)
- X-Content-Type-Options: nosniff
- X-XSS-Protection: 1; mode=block
- Referrer-Policy: strict-origin-when-cross-origin
- Permissions-Policy (geolocation, microphone, camera)
- Strict-Transport-Security (production HTTPS)

8.4 Rate Limiting:
- File-based rate limiting system
- 60 requests per minute per IP (configurable)
- Login-specific rate limiting (10 per 5 minutes)
- HTTP 429 response with Retry-After header

8.5 File Security:
- Randomized upload filenames (prevents enumeration)
- PHP content detection in uploads (blocks malicious files)
- .htaccess blocks access to config/, includes/, .env, .git
- Directory listing disabled (Options -Indexes)
- Sensitive file types blocked (.sql, .env, .log, etc.)

10. PAYMENT INTEGRATION
================================================================================

Supported Payment Methods:
1. Cash on Delivery (COD) - Default
2. JazzCash (Pakistan's leading mobile wallet)
3. EasyPaisa (Telenor's mobile payment)
4. Bank Transfer
5. Credit/Debit Card

Payment Flow:
1. Customer books a service
2. Selects payment method
3. For digital payments, redirected to payment gateway
4. Gateway processes payment and sends callback
5. System records transaction and updates booking status

Note: Currently running in sandbox/demo mode. Real gateway credentials
can be configured in the .env file for production deployment.

Commission System:
- Platform charges a configurable commission (default 15%)
- Commission rate configurable in admin settings
- Technician receives (100% - commission) of booking amount

  /login               -> Login page

  /logout              -> Logout action
  /forgot-password     -> Password reset

  /user/dashboard      -> Customer dashboard
  /user/bookings       -> Customer bookings
  /technician/dashboard-> Technician dashboard
  /technician/bookings -> Technician bookings
  /admin/dashboard     -> Admin dashboard
  /admin/users         -> User management

  /api/auth            -> Authentication API
  /api/search          -> Search API
  /api/notifications   -> Notifications API
  /api/payment/*       -> Payment callbacks

- router.php for PHP built-in server (development)
